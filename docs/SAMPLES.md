# NICO Samples and Fixtures

NICO keeps sample evidence separate from live production claims. Synthetic fixtures, recorded demonstrations, and workflow-generated samples must remain labeled and must not be represented as client assessments, approvals, certifications, or deployed repairs.

## Synthetic golden fixtures

The canonical synthetic fixture manifest is:

```text
tests/fixtures/golden/manifest.json
```

It covers:

- unavailable scanner evidence with no passing credit;
- complete evidence that still requires human review;
- mixed-risk findings with bounded repair candidates, verification, and rollback;
- delivery blocked until explicit approval;
- no certification or automatic production-change claim.

The `Recorded Golden Demonstration` workflow builds deterministic JSON and Markdown artifacts from these fixtures and verifies reproducibility.

## Express sample artifact

The manual `Express Sample Artifact` workflow runs an authorized local assessment against the checked-out repository and uploads a structured sample bundle containing:

- JSON, Markdown, and HTML assessment outputs;
- evidence manifests;
- executive and technical summaries;
- dependency, CI/CD, architecture, activity, and maturity evidence;
- a 30/60/90 roadmap, resourcing plan, and limitations.

Run it from GitHub Actions using `workflow_dispatch`. The uploaded artifact is evidence from that workflow execution, not a permanent public client sample.

## PDF samples

PDF output is generated by the report pipeline where supported, but the repository does not commit a binary PDF as authoritative production evidence. Binary samples become stale quickly and can be mistaken for live or approved results.

For demonstrations:

1. generate a fresh sample from a known synthetic or authorized target;
2. retain the source commit and workflow/run identity;
3. label the output synthetic or demonstration-only;
4. preserve `human_review_required: true` and `client_ready: false` unless a separate approval artifact exists;
5. do not publish secrets, private repository content, client data, delivery tokens, or administrator credentials.

## Local sample generation

```bash
rm -rf assessment_latest
python -m nico.assessment . --tier express --output assessment_latest
```

Expected output includes:

```text
assessment_latest/assessment_latest.json
assessment_latest/assessment_latest.md
assessment_latest/assessment_latest.html
assessment_latest/evidence_manifest.json
assessment_latest/express_pack/
```

Review the generated limitations and unavailable evidence before sharing any output.

## What samples do not prove

Samples and fixtures do not prove:

- a deployed Express, Mid, or Full browser/API run completed correctly;
- a scanner found every relevant issue;
- a score is a security certification;
- a repair was implemented or verified in production;
- a report is approved or client-ready;
- hosted persistence, recovery, or delivery survived a real outage.
