# Score-to-85 Next Steps

Current target: move NICO Express from 77 toward 85 using evidence-backed improvements only.

## PR #39 scope

Artifact access diagnostics and report-visible artifact-access blockers.

This PR does not claim scanner artifacts are clean when access is missing. Instead, it makes the blocker visible in diagnostics and reports so the deployment can be configured correctly.

## Expected effect

- If `scanner_artifacts.status` is `token_missing`, configure `NICO_GITHUB_TOKEN` or `GITHUB_TOKEN` in the backend deployment and rerun the report.
- If `scanner_artifacts.status` is `ok`, NICO can attempt to credit current scanner artifacts.
- If scanner artifacts are clean, Secrets, Static Analysis, Dependency, and CI/CD sections can improve.
- If artifacts contain findings, the report should show the findings instead of raising the score.

## Follow-up PRs

1. Commit a real `apps/web/package-lock.json` generated by `npm install` in `apps/web`.
2. Add gitleaks or trufflehog JSON artifact generation.
3. Parse scanner severity levels from Bandit/Semgrep instead of only counting findings.
4. Add pytest coverage artifacts and score test coverage evidence.

## Rule

Do not raise scoring caps without stronger evidence. The path to 85 is artifact access, clean scanner evidence, reproducible dependencies, and test coverage evidence.
