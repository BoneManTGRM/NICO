# NICO Mid Evidence Packet

This directory contains version-controlled context that can be attached to a snapshot-bound Mid Assessment for human review.

- `ARCHITECTURE.md` — intended system structure and trust boundaries.
- `DEPLOYMENT.md` — hosted frontend/backend configuration and runtime validation steps.
- `QA.md` — automated and manual QA boundaries.
- `PRODUCT_CONTEXT.md` — product purpose, rules, intended user outcomes, and current constraints.
- `ROADMAP.md` — remediation priorities and Issue 296 completion criteria.

These documents are contextual evidence, not automatic proof. They must not change scores without human validation. Runtime, scanner, deployment, build, stakeholder, and platform claims still require direct evidence from the exact assessment scope.
