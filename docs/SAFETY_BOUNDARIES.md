# NICO Safety Boundaries

NICO is defensive-only and may operate only on user-authorized local repositories, test labs, owned applications, authorized logs, authorized cloud accounts, or authorized security tools.

Blocked categories:

- unauthorized scanning
- exploitation of third-party systems
- credential theft
- phishing
- malware
- stealth
- evasion
- persistence
- destructive activity
- authentication bypass
- offensive attack automation
- raw secret exposure

High-blast-radius production actions require human approval.
